📢 Live In-App Banner

Writes to notifications/latest. All users on the site see the banner immediately via Firestore onSnapshot.

📊 Live Fetch Progress

Controls the progress bar on simulation.html via notifications/fetch_progress. Update percent, counts, status (running / paused / completed), and timing fields manually while data is being fetched.

📅 Induction Schedule

Publishes portal step windows to the Schedule tab on simulation.html via notifications/induction_schedule in Firestore. Do not fetch the portal API from this public site — requests from GitHub Pages are blocked and would expose the private endpoint. Fetch the schedule locally (browser on the portal while logged in, Postman, etc.), paste the JSON Table array or full response below, then save.

✅ Profile Verification Status

Publishes per-applicant verification status to the Candidates tab on simulation.html via notifications/profile_status in Firestore. Paste the portal ProfileStatus array (or full response with entries) — same workflow as schedule: fetch locally, paste JSON, save. Status codes: 11 pending · 1 accepted · 2 rejected.

📌 Community Chat — Pinned Announcement

Shows a pinned banner at the top of community chat on simulation.html via notifications/chat_pin. Only one announcement at a time — use for rules, deadlines, or important updates.

📝 Simulation Notifications Feed

Edits the simulation page notification feed using the same schema as data/notifications.json. On GitHub Pages this is stored in notifications/simulation_feed in Firestore, because the admin portal cannot directly rewrite static repo files from the browser.

📊 Simulation Merit Formulas

Configure how base merit marks are calculated on the simulation page (candidate pool, slot browser, and allocation). Stored in notifications/marks_config. Users pick from these options unless you hide the selector. Use adjustments when portal marksTotal includes or excludes components inconsistently (e.g. MDCAT). Per-programme dict fields: programAttempt (FCPS), programPercentage (MD/MS), programMarks, and adjusted (all programmes). Bare names use the active simulation programme; dotted keys like programPercentage.MD target a specific programme. Missing subkeys resolve to 0. Adjustment fields can be typed or picked from suggestions.

Candidate notice (simulation page)

Shown as a highlighted banner near merit filters so candidates understand how marks are calculated.

🧮 Seat Allocation Status Scope

Controls which profile verification statuses can be included in the Seat Allocation simulation. Stored in notifications/simulation_config. Hide the selector to force one admin-chosen scope for everyone.

Add custom status scope

Common verification status IDs: 1 Accepted, 11 Pending, 2 Rejected. You can combine any IDs present in the imported profile-status data.

🔐 Access Request & Payment Config

Controls the request access page, auth gate request tab, and payment/support block. Stored in notifications/access_config. Account title and extra payment IDs are not shown to candidates.

📱 Web Push Subscribers

Users who clicked "Alerts" → allowed browser notifications. Their FCM tokens are stored in push_subscriptions. To send a push to closed browsers, use Firebase Console → Cloud Messaging.
